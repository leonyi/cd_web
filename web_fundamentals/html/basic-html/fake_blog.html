A Cheat Sheet

Action Command Comments
Create a new repository git init
Check out a repository git clone username@host:/path/to/repository
Add files git add your_filename or git add username@host:/path/to/repository
Commit files git commit -m "My commit message"
Push files git push origin master To connect your existing repo to a remote server add it with git remote add origin
Update files git pull This fetches and merges remote changes.
Merge files git merge other_branch This may result in conflicts that you will have to manually resolve. After resolving do a "git add your_filename" to mark them as merged.
Remove Branches git branch -d branch_name (local branch) To remove a remote branch run, git push :branch_name

Branching

Branches are used to develop features and keep them isolated from each other. The master branch must contain the code that is currently running in production. Other branches are used for active development and testing. Once they are ready they can be merged to the master branch. To create a branch we do git checkout -b new_feature_branch

Replacing Local Changes

If something goes awry you can replace local changes with git checkout -- your_filename
